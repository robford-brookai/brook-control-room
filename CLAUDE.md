## Project Overview

Brook Control Room — a healthcare analytics dashboard tracking patient funnel metrics, care delivery, revenue, and program adoption for RPM (Remote Patient Monitoring), CCM (Chronic Care Management), and APCM (Annual Personalized Preventive Care) programs.

Real data: 205 days of daily funnel snapshots (Sept 2025–Mar 2026). Synthetic extensions for monitoring rates, care codes, revenue, and program distribution are generated by a Python script with seeded randomness.

## Commands

**Frontend:**
```bash
npm run dev       # Start Vite dev server → http://localhost:5173
npm run build     # Build production dist/
npm run preview   # Preview production build
```

**Data pipeline:**
```bash
python3 scripts/generate-data.py   # Regenerate src/data.json and src/daily-detail.json from CSV
```

```bash
uv run ruff check scripts/   # Lint

```

## Architecture

Static frontend — no backend or API. Data flows CSV → Python → JSON → React.

```
data/daily-funnel-data.csv (205 rows, 37 cols)
        │
        ▼
scripts/generate-data.py
        │
        ├─► src/data.json         (13.5KB aggregated: monthly, march_daily, pacing, funnel, churn, targets)
        └─► src/daily-detail.json (211KB full daily records for drill-down)
                │
                ▼
        src/BrookControlRoom.jsx  (main dashboard, 703 lines, 6 tabbed phases)
```

**Stack:** React 18 + Vite 5, Recharts 2.12 (charts), Lucide React (icons), inline CSS only (dark theme).

## Key Files

| File | Role |
|------|------|
| `src/BrookControlRoom.jsx` | Entire dashboard — 6 phases, reusable UI components, color constants |
| `src/data.json` | Aggregated data consumed by dashboard |
| `src/daily-detail.json` | Full daily records (do not edit manually) |
| `scripts/generate-data.py` | Source-of-truth for all data transformations and synthetic extensions |
| `data/daily-funnel-data.csv` | Raw source data — the only file to edit for real data updates |

## Dashboard Phases

- **P1 Control Room**: KPI tiles, pacing curves, daily activations
- **P2 Growth Pipeline**: Acquisition funnel, gap/leakage analysis, device pipeline
- **P2 Care Delivery**: Monitoring rate, program splits, revenue/code breakdown
- **P2 Programs & Retention**: Cohort adoption trends, churn drivers
- **P3 Revenue Analytics**: Tier breakdown, net revenue trend
- **PN Future Metrics**: Placeholder

## Data Model

**Reimbursement rates** (in generate-data.py): RPM $62.50, CCM $48.00, APCM $85.00

**March 2026 targets**: Activations 653, Active patients 8,584, Care codes 5,200, Revenue $380K, Monitoring 80%

**Synthetic generation rules:**
- Monitoring rates: 75–85% with weekday/weekend split (79% / 55%)
- Care code accumulation: S-curve, back-loaded
- Program mix: HTN 55%, DM 30%, BH 10%, DPP 5%
- Seeded randomness: `random.seed(42)` — reproducible output

2. Run `python3 scripts/generate-data.py`
3. Vite hot-reload picks up new JSON automatically
